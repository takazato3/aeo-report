# Ops Octopus LP Design Specification

## 0. このドキュメントの目的

このドキュメントは、`opsoctopus / Ops Octopus` のWebサイト（LP）を全面刷新するためのデザイン・実装指示書です。

現在のデザイン（色・フォント・レイアウト）は一旦忘れ、ゼロベースで制作してください。

ただし、以下の方針は必ず守ってください。

- 何のサービスかが3秒で分かる
- かわいいタコではなく、レポート成果物が主役
- ヒーローは尖っていてよいが、下層は信頼感を優先
- レポートサンプルをファーストビューまたは上部に必ず見せる
- 情報商材っぽく見せない
- 大企業SaaS風に背伸びしすぎず、スモールSaaSとしての個性を残す
- Quick Scanは気軽に買えるが、Deep / Fullの将来価値も伝わる
- 「AEOを知らないが気になっている人」にも分かる言葉で書く
- 「面倒な調査代行」は後段で活かし、ヒーロー主コピーにはしない
- LP上のタコは案内役、レポート上のタコは極力抑える

## 2. ブランド情報

### ブランド名

`opsoctopus / Ops Octopus`

### ロゴ表記

ロゴまわりは以下を近い位置にまとめてください。

- タコアイコン
- `Ops Octopus`
- `AIブランド調査レポート`
- できれば小さくブランドコピー：`『知りたい』に、するっと。`

初見でも「何のサービスか」が分かるよう、ロゴ、説明肩書き、ブランドコピーを分散させず、ひとつのロックアップとして見せてください。

### ブランドコピー

```text
『知りたい』に、するっと。
```

必須ではありませんが、できるだけロゴまわりに小さく添えてください。

### マスコット

マスコットは、ダークネイビーのにらみタコです。

特徴：

- ダークネイビーのタコ
- イエロー / アンバーの目
- 少し尖っているが、愛嬌もある
- かわいすぎない
- 過剰にポップにしない
- LP上では“案内役”として使う

### タコキャラクターの扱い

タコキャラクターはLP上ではブランド記憶のために使ってよいです。  
ただし、**出力レポートの見た目では強く出さない** でください。

レポートサンプル上では、以下のどちらかにしてください。

- タコを一切出さない
- テキストロゴに小さなアイコンシルエットを被せる程度に抑える

成果物は「業務資料として使えそう」に見えることを優先します。

## 3. デザイン全体の方向性

### 全体コンセプト

```text
ダークで尖った入口、白く読みやすい成果物、実務サービスとしての信頼感
```

### 方向性

- 尖ったスモールSaaSのサービスサイト
- クールだが無機質ではない
- 個人開発・スモールビジネスらしい軽やかさは残す
- ただし、情報商材っぽさ・怪しさは徹底的に避ける
- 300円のお試しは気軽に買える
- 2,980円 / 9,800円プランは、レポート商品として信頼できる見た目にする

### レイアウトの基本方針

- ヒーローのみ深いネイビー / ディープブルーグレー背景
- 下層セクションはオフホワイトまたは薄いブルーグレー
- レポートサンプル部分は白背景で“資料感”を出す
- CTAや強調にイエロー / アンバーを使う
- テキストはヒーローではオフホワイト、下層では濃いネイビーまたはチャコール
- 全面ダークモードにはしない
- 読み物・成果物・料金表は明るい背景で信頼感を出す
- アイコン類はGoogle Material Icons（CDN）を使用
- ChatGPT・Claude・GeminiなどのブランドロゴはMaterial Iconsの汎用アイコンで代替

### カラーイメージ

| 用途 | 色の方向性 |
|---|---|
| ベース | ダークネイビー、ディープブルーグレー |
| 下層背景 | オフホワイト、薄いブルーグレー |
| アクセント | イエロー / アンバー |
| テキスト | オフホワイト、濃紺、チャコール |
| カード | 白背景＋薄い罫線 |

参考値：

```css

  --navy-800: #102a4a;
  --blue-gray-50: #f4f7fb;

  --amber-500: #f59e0b;
  --text-dark: #132033;
  --text-muted: #5f6f85;
  --border: #dce4ef;
}
```

### フォント

- 日本語表示に対応すること
- モダンで読みやすく、実務サービスとしての信頼感を損なわないこと
- 見出しはややシャープでSaaSらしい印象を持たせること
- 本文は長文でも読みやすい可読性を優先すること
- 画面表示用フォントと、HTMLダウンロード / PDF出力用フォントは分けて定義すること

#### 画面表示用フォント

```css
font-family:
  "Inter",
  "Noto Sans JP",
  "Hiragino Sans",
  "Yu Gothic",
  sans-serif;

- Inter を英数字・UIの主軸として使用
- Noto Sans JP を日本語表示の主軸として使用
- Hiragino Sans / Yu Gothic はローカル環境向けのフォールバックとしてのみ使用

#### エクスポート用フォント（HTMLダウンロード / PDF出力）

```css
font-family:
  "Inter",
  "Noto Sans JP",
  sans-serif;

- ダウンロードHTMLおよびPDF出力では、配布・埋め込み可能なフォントのみを使用すること
- Hiragino Sans、Yu Gothic などのシステム依存フォントを、エクスポート成果物の前提フォントにしないこと
- PDF生成時にフォント埋め込みが発生する可能性を考慮し、権利と再現性が明確なフォント構成に限定すること

外部フォントを使う場合は、商用利用可能なものに限定してください。  
権利不明の有料フォントや、ライセンス確認できないフォントは指定しないでください。

## 4. ヒーローコピー

ヒーローの主コピーは以下を最優先で使用してください。

```text
AEO、気になる。
でも、何を見ればいいのか分からない。
```

「めんどくさい調査を代行する」訴求は重要ですが、ヒーロー主コピーには置きません。  
理由は、代行訴求は「すでに知りたいことが分かっている人」には刺さる一方、ヒーローに置くと間口が狭くなるためです。

### ヒーローのサブコピー例

```text
ChatGPT・Claude・Geminiに同じ条件（プロンプト）で問い合わせ、
AI上でのブランドの推奨状況をレポート化します。
まずは300円のお試し調査から。
```

### CTA例

- `Quick Scanを試す（\300）`
- `サンプルレポートを見る`

### ヒーロー上の必須要素

ファーストビュー内に、白背景のレポート画像モックをはめ込んでください。  
ユーザーがページを開いた瞬間に「購入後にどんな成果物が見られるのか」を概観できるようにします。

# 6-2. 課題提起

## 目的

「AEOが気になるけど、何から手をつけていいかわからない現場担当者」の課題感を表現してください。

## トーン

少し口語調。  
ただし砕けすぎず、実務者が自分ごと化できる温度にします。

## 入れたい内容

- AEO / LLMO / AI検索が気になる
- でも、何を調べればいいか分からない
- 自社名をAIに聞いてみても、1回の回答だけでは判断できない
- ChatGPTだけ見ても、ClaudeやGeminiでは違う可能性がある
- 自分が使い込んでいるAIでは、偏った回答をするのではないか
- 競合と比べたいが、どう調べればいいか分からない
- 毎回手作業で聞いて、見比べて、まとめるのは面倒

## コピー例

```text
こんな悩み、ありませんか？
```

箇条書き例：

```text
AEO / LLMO / AI検索が気になるけど、何から手をつけていいか分からない
自社名をAIに聞いてみても、1回の回答だけでは判断できない
ChatGPTだけ見ても、ClaudeやGeminiでは違う可能性がある
競合と比べてどうなのかも、パッと分かりにくい
毎回手作業で聞いて、見比べて、まとめるのは正直しんどい
```

必要に応じて、タコの小さなイラストを案内役として使ってよいです。

# 6-4. プロダクト説明

## 目的

3AIに実際に問い合わせ、ブランドのAI上での推奨状況をレポートにするサービスであることを端的に説明してください。

## 入れたい要素

- ChatGPT / Claude / Gemini の３AIに横断で問い合わせ
- 同じ条件で調査
- AI上でブランドがどう推奨・言及されているかを見る
- 1回きりの回答ではなく、プランに応じ最大90回（3AI×30回)調査する
- 結果を見やすいレポートとして提供する

## コピー例

```text
3つのAIに聞いて、ブランドの見え方をレポート化します。
```

## 説明コピー例

```text
OpsOctopusの`AIブランド調査レポート`は、ChatGPT・Claude・Geminiの3つのAIに同じ条件で問い合わせ、
ブランド名やサービス名、関連キーワードがAI上でどのように推奨・言及されているかを調査します。
結果は見やすいレポートとして整理し、施策前の現状把握に使える判断材料として提供します。
```

# 6-9. FAQ / 注意事項

フッター前に短いFAQを入れてください。  
信頼性と不安解消のためのセクションです。

## FAQに入れたい内容

### Q. AIの回答は変動しますか？

A. はい。AIの回答は、調査時点、モデル更新、入力条件などによって変動する可能性があります。本サービスのレポートは、調査時点のスナップショットとしてご利用ください。

### Q. 調査結果は成果を保証しますか？

A. いいえ。本サービスはAI検索エンジンの回答傾向を調査・集計するものであり、検索順位の改善や特定成果を保証するものではありません。

### Q. ChatGPT / Claude / Geminiの公式サービスですか？

A. いいえ。本サービスは各社が公式に提供・認定するものではありません。

### Q. 入力したブランド名やキーワードはどう扱われますか？

A. 調査レポート作成のために使用します。詳細はプライバシーポリシーを確認ください。

### Q. Deep Scan / Full Scanはいつ使えますか？

A. 現在準備中です。サンプルレポート公開後、順次受付予定です。

## 7. レポートモックのデザイン方針

### 方針

- 白背景
- 薄い罫線
- 表・カード・グラフを中心にする
- 業務資料として使えそうに見せる
- 過剰に派手な色を使わない
- タコキャラクターは強く出さない
- レポート名や小さなロゴ程度に留める

- エグゼクティブサマリー

## 9. テキストトーン

### 基本

- やさしいが、幼くしない
- 実務者向けに、分かりやすくする
- AEO初心者でも分かる
- ただし、説明しすぎて冗長にしない
- 「すごい」「稼げる」「絶対」などの煽りは避ける

### 避ける表現

- 成果保証に見える表現
- 過度な不安訴求
- 情報商材っぽい煽り
- 大企業SaaS風の大げさすぎるコピー
- タコを前面に出しすぎる遊びすぎた表現

### 使いたい表現

- 現状把握
- 判断材料
- 傾向を見る
- レポート化
- 買い切り
- サブスク不要
- 営業なし
- まず試す
- 3AI横断
